# Общее
Если работаешь с github, то всегда, когда возможно, предпочитай использовать github mcp.

## Получение информации
Используй "github mcp" вместо "web content fetch" для получения информации о github проекте.

## Создание Pull Request
Для создания pull request нужно выполнить следующие действия:
1. создать новую ветку, выполнить checkout на нее;
2. закоммитить изменения, относящиеся к pull request;
3. выполнить git push с указанием remote и названия ветки;
4. создать pull request из созданной ветки в необходимую.
